## 🔍 问题描述

大家在玩 **VMware虚拟机** 的时候，发现虚拟机非常卡顿，甚至有时候鼠标都卡顿，键盘输入也很慢，岂可修啊！！明明已经提高了 **内存** 和 **处理器个数**，怎么还是这么卡

---

## 💡 解决方案

### 步骤一：启用虚拟化引擎

其实把VMware的虚拟化 `AMD-V/RVI` 打开就行了，大概在：

> **虚拟机设置** → **处理器** → **虚拟化引擎**

勾选上就行了。

::: warning
当你信心满满地打开虚拟机之后，他可能显示了 **"此平台不支持的虚拟化AMD-V/RVI"**，最终启动失败了
:::

::: tip
你的 **Windows** 不是家庭版，而是 **专业版** 或者其他的订阅版本
:::

---

### 步骤二：关闭核心隔离相关功能

#### 📍 操作路径

进入：**设置** > **隐私与安全** > **Windows 安全中心** > **设备安全性** > **核心隔离**

#### 🔧 需要关闭的选项

- **内存完整性**（Memory integrity）
- **固件保护**（Firmware protection）

---

### 步骤三：关闭 Hypervisor

#### 1️⃣ 运行命令

**管理员模式** 运行 **命令提示符**，运行下面命令并重启：

```bash
bcdedit /set hypervisorlaunchtype off
```

::: tip
这条命令有可能没有效果，你需要参考这边文章：[https://blog.csdn.net/liuzhuo13396/article/details/148693513](https://blog.csdn.net/liuzhuo13396/article/details/148693513),采用组策略方案即可
:::

#### 2️⃣ 验证结果

重启系统后，按下 `Windows + R`，输入 `msinfo32`，点击确定。

::: tip
如果你没有显示关于**Hyper-V**的信息，你需要在启动Windows功能去启用一下。
:::

在 **系统信息** 中应看到如下结果：

| 项目 | 状态 |
|------|------|
| **内核DMA保护** / Kernel DMA Protection | ✅ **关** / Off |
| **基于虚拟化的安全性** / Virtualization-based security | ✅ **未启用** / Not enabled |

---

## 📝 总结

遇到 **VMware虚拟机** 卡顿的问题，可以尝试以下方法：

1. ✅ 启用 `AMD-V/RVI` 虚拟化引擎
2. ✅ 关闭核心隔离功能
3. ✅ 关闭 Hypervisor 功能
